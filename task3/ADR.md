### <a name="_3bfxc9a45514"></a>**Функциональные требования**
1. Клиент может просматривать список доступных депозитов
   с актуальными и персонализированными ставками.
2. Клиент может подать заявку на депозит, указав ФИО и телефон.
3. Менеджер колл-центра обрабатывает заявки, может предложить персональные условия.
4. Клиент интернет-банка может подавать заявки с указанием счета и суммы, подтверждая через СМС.
5. Клиент может просматривать список доступных депозитов и актуальные ставки.

Опишите здесь наиболее важные альтернативные решения.

**Недостатки, ограничения, риски**
Человеческий фактор:
Расчёт персональных ставок вручную через Excel может привести к ошибкам.
Нагрузка на АБС:
Прямое подключение интернет-банка к АБС ограничено, чтобы не перегружать базу данных.

Ограничения MVP:
Новый клиент обязательно должен проходить идентификацию в отделении.
Полная автоматизация и интеграция всех продуктов банка пока невозможны.

Риски безопасности:
Необходимо корректно шифровать все чувствительные данные при передаче между системами.

Ограничения технологического стека:
Используются только технологии, с которыми есть экспертиза (MS SQL, Oracle, существующие внутренние сервисы).
Переход на микросервисную архитектуру ограничен только процессом открытия депозитов.
